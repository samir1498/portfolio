The **context system** is a repo of markdown files with structured YAML frontmatter — plans, roadmaps, research references, and progress logs — that AI coding agents read and write as they work. `pc-ctx` is the CLI, MCP server, and web UI that wrap it.

## What changed

Over about three weeks, it went from "a note asking agents to read some markdown files" to:

- **`@pc-ctx/core`** — Types, YAML I/O, plan CRUD, storage backends
- **`@pc-ctx/cli`** — `ctx plan add`, `ctx status`, `ctx graph`, `ctx research`, `ctx sync`
- **`@pc-ctx/mcp`** — MCP server wrapping all CLI commands
- **Web UI** — Dashboard, plans list, folder pages, all behind Cloudflare Access

## Architecture

```
┌─────────────┐     ┌──────────────┐     ┌────────────┐
│  CLI (citty)│────▶│  @pc-ctx/core│────▶│  Markdown  │
│  MCP server │     │  + js-yaml   │     │  + Git     │
├─────────────┤     └──────────────┘     └────────────┘
│  Web UI     │
│  Hono SPA   │────▶ GitHub API (authenticated)
│  React 19   │
└─────────────┘
```

## The Web UI

The web interface is a single Hono Pages Function serving a React 19 SPA:

- **Dashboard**: Animated SVG donut chart for status distribution, animated bar chart for weekly activity, count-up number counters, priority-sorted top plans, recent progress timeline — all with framer-motion staggered entrance animations
- **Plans page**: Full-text search, status filter, sort by priority/date/name, pagination (10/page), HoverCard summaries on each row
- **Folder pages**: Shared generic component for roadmaps, references, and progress — each with inline markdown rendering on click
- **Plan detail**: Rendered markdown with syntax highlighting (highlight.js), task list with status badges, priority and metadata
- **Responsive**: Side-by-side layout on desktop, stacked on mobile
- **Auth**: Cloudflare Access at the edge — zero code, private by default

### Stack

| Layer      | Choice                                   |
| ---------- | ---------------------------------------- |
| Runtime    | Cloudflare Pages (Hono Functions Worker) |
| Frontend   | React 19 + Vite + Tailwind CSS v4        |
| Routing    | TanStack Router (file-based)             |
| Data       | TanStack Query (30s stale, API cache)    |
| Charts     | Pure SVG (donut) + animated bars         |
| Animations | framer-motion                            |
| Auth       | Cloudflare Access (edge, zero config)    |
| Deploy     | `wrangler pages deploy`                  |
| Local dev  | `bun run ctx ui`                         |
