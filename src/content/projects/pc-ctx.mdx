  - "16 MCP tools: plan_list, plan_show, plan_status, plan_validate, plan_set_status, plan_task_status, plan_add, plan_add_task, plan_references, roadmap_list, roadmap_show, research_list, research_show, graph, sync, setup"

  - "Roadmaps that reference plans, so agents know the big picture"
  - "Research files indexed from linked repos, browsable from the CLI"

  - "Git-backed. Agents commit their own progress."
  - "GitHub sync — `ctx sync` pushes and pulls context across machines"

highlights:
  - title: "Scope Creep, but Make It Useful"
    description: "It started as markdown files and a note saying 'agents should read these'. Then a CLI so they would not parse YAML themselves. Then an MCP server so they would not parse CLI output either. What was a single script became three npm packages."
  - title: "Built for Agents, Not Dashboards"
    description: "No UI, no database, no auth. Just markdown files with structured frontmatter in a git repo. The CLI and MCP server exist so agents can manipulate them deterministically."
  - title: "16 MCP Tools, Zero Adapters"
    description: "OpenCode, Claude Code, Cursor, Windsurf. If it speaks MCP, it reads and writes plans. No adapter per agent. No CLAUDE.md hacks. 16 tools covering plans, roadmaps, research, graph, and sync."
  - title: "Install Any Way You Like"
    description: "`npm install -g @pc-ctx/cli`, `pnpm add -g @pc-ctx/cli`, or `bun add -g @pc-ctx/cli` — pick your package manager. Then `ctx setup ./project` creates the whole tree, writes templates, installs deps, and inits git."
  - title: "Format Is the Product"
    description: "Tasks have IDs and statuses, acceptance criteria are checkboxes, everything has a schema. Agents can't drift, the CLI validates before every read."
