In my current role, I work across different technologies. Outside work, Java is the language I want to master deeply. This year, I am making that commitment explicit.

I have used Java before, but I want a stronger foundation at the level that matters in production: runtime behavior, memory, concurrency, and performance. Not just syntax.

## What "Deep" Means For Me

- Understand Java evolution from **8 to 25** with engineering impact, not only feature lists.
- Study JVM internals: class loading, JIT compilation flow, and garbage collection trade-offs.
- Practice concurrency with modern Java patterns, including virtual threads.
- Use profiling and benchmarking tools to validate performance assumptions.

## Execution Plan

I am starting with **1 hour per day** and focusing on consistency:

- Reading and notes
- Hands-on experiments
- Weekly summary of what worked and what did not

## Public Commitment

I will use this blog to publish concise updates: lessons learned, experiments, and practical takeaways from each step. The goal is simple: measurable progress, week after week.
